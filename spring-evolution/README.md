# User Reminder Service --- проект, демонстрирующий разные версии Spring (1.x --> 7.x)

Этот проект демонстрирует, как одна и та же предметная область и один и
тот же REST-контракт могут быть реализованы в разных поколениях Spring
Framework: от Spring 1.x (XML, BeanFactory, Acegi) до Spring 7.x
(Virtual Threads, HTTP Interface Clients, Resilience API).

Цель - показать, **как менялись подходы, структура и архитектура
приложений вместе с развитием Spring**, не изменяя самой
бизнес-функциональности.

------------------------------------------------------------------------

## Запуск
В каждой папке есть docker-compose . Для запуска проекта нужно зайти в папку проекта и выполнить команду:
```
docker-compose build
docker-compose up
```
Для отправки http-запросов можно использовать файлы postman-spring-*.json . 
Они расположены в корне каждого проекта на одном уровне с docker-compose файлами.
Файлы postman необходимо импортировать в postman. После чего можно отправлять запросы: все необходимые параметры для запроса и секюрити уже прописаны.

------------------------------------------------------------------------

## Идея приложения

Простое приложение для управления напоминаниями пользователей:

-   Пользователь (`User`) может быть создан.
-   Для пользователя можно создать напоминание (`Reminder`) - текст +
    дата/время.
-   При создании напоминания система отправляет уведомление (в разных
    версиях может немного отличаться).
-   Приложение использует PostgreSQL.
-   Доступ к API защищён простой базовой аутентификацией (Acegi/Spring
    Security).

------------------------------------------------------------------------

Этот проект помогает увидеть эволюцию Spring Framework от ранних версий
до современных подходов, применимых в cloud-native и high-load системах.
